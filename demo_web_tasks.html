🚀 Active Task Simulator

This page helps demonstrate the Active Task Tracking feature.

⚠️ Note: The task tracking system stores tasks in the Flask process memory. To see active tasks, they must be added from within the Flask application context.

How it works in production:
  • When you run a backtest, it calls add_task('Backtest', 'backtest', 'details')
  • Progress updates are sent with update_task(task_id, progress=50)
  • When complete, mark with update_task(task_id, status='completed')

🎯 Integration Example

Here's how to integrate task tracking into your trading modules:

# In your trading module (e.g., backtester.py, simulated_live_trading.py)
from dashboard import add_task, update_task, remove_task

def run_backtest(strategy, data):
    # Add task when starting
    task_id = add_task(
        task_name=f"Backtest - {strategy.name}",
        task_type="backtest",
        details=f"Testing on {len(data)} candles"
    )
    
    try:
        # Update progress as you go
        for i, candle in enumerate(data):
            process_candle(candle)
            progress = int((i / len(data)) * 100)
            update_task(task_id, progress=progress)
        
        # Mark as completed
        update_task(task_id, status="completed", 
                   details="Backtest completed successfully")
    except Exception as e:
        # Mark as failed
        update_task(task_id, status="failed", 
                   details=f"Error: {str(e)}")
